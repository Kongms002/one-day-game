# 보스 스킬 설계안 및 설정 스키마

## 1) 핵심 컨셉
보스 스킬은 `SkillDefinition`이 아니라 `Action Block`의 조합으로 본다.

- 보스는 스킬을 직접 실행하지 않고 `BossSkillExecutor`에게 위임
- 스킬 실행은 구성요소가 순차/병렬로 조합된 `SkillPipeline`
- 스킬은 1개 스킬 안에 투사체+영역+데미지+상태이상+재생성 루틴을 모두 담을 수 있음

## 2) 스킬 구성 계층
### 2.1 BossSkillSO
- `skillId`, `displayName`, `description`, `cooldown`, `weight`, `castTime`, `windupTime`, `channelTime`
- `targeting`: `ITargetingPolicy` 참조
- `effects`: `ISkillEffect` 리스트
- `selectors`: 보너스 타깃/범위/조건 선택기
- `constraints`: `isInterruptible`, `isUsableDuringPhase`, `canOverlapWithOthers`

### 2.2 Effect 타입(샘플)
- `DamageEffectSO`
  - `amountType`(flat/multiplier), `damageType`, `critChance`, `ignoreShield`, `isLethal` 제약
- `ProjectileEffectSO`
  - `count`, `spread`, `speed`, `lifeTime`, `pierceCount`, `onHitPolicy`
- `AreaDamageEffectSO`
  - `areaShape`, `radius`, `tickRate`, `duration`, `maxTargets`
- `StatusEffectSO`
  - `stun`, `slow`, `poison`, `burn`, `stunImmunity`
- `SummonEffectSO`
  - minions/pet/spawnCount/spawnPointRules
- `ForceEffectSO`(optional)
  - 넉백, 띄우기, 플레이어 끌어당김 등

### 2.3 Targeting/선정 계층
- `ITargetingStrategy`
  - `SelectPrimaryTargets(ctx)`
- 구현 예시
  - `NearestNStrategy`
  - `ClosestByThreatStrategy`
  - `CircleAreaStrategy`
  - `ConeAreaStrategy`
  - `RandomizedWeightedStrategy`
- 대상이 없을 경우 `FallbackStrategy`로 무해 실패 처리 (NPE/InvalidAccess 방지)

### 2.4 실행 정책
- `ISkillExecutor`
  - `ExecuteAsync(ctx)` 단일 진입점
- `IExecutionOrderPolicy`
  - 순차/병렬/조건부 분기 처리
- `ISkillCooldownPolicy`
  - 보스별 쿨다운 보정, phase 기반 multiplier

## 3) 보스 설정(권장) 
### BossConfigSO
- `bossId`, `name`, `prefab`, `entryStats`, `deathRewards`
- `phases`: `BossPhaseSO[]`
- `resistances`, `weaknesses`, `killTriggers`

### BossPhaseSO
- `phaseIndex`, `hpRange`(min,max)
- `musicBgm`, `introSfx`, `phaseTransitionAction`
- `skillWeightTable`: Phase별 선호 스킬 가중치 맵
- `aiModifiers`: 공격속도/회피/방어 보정치
- `skillBlacklist/allowlist`

### BossSkillSetSO
- 보스 기본 스킬셋 + phase별 추가/금지 스킬 목록
- 기본 2~3개 이상 패턴을 기본값으로 운용

### SpawnConfigSO (선택)
- 보스 등장/재생성용 스폰 수치
- 피격 판정 오브젝트 범위, 카메라 쉐이크, 연출 트리거

## 4) 상호작용 시퀀스(요약)
1. `BossAI`가 `BossRunContext` 기반으로 phase/상태 계산
2. `BossSkillSelector`가 페이즈+쿨다운+거리 조건으로 후보 필터링
3. `WeightedSkillSelector`가 가중치 기반 스킬 선택
4. `Targeting` 실행 후 target 집합 생성
5. `SkillPipeline` 실행 (`Effect`별 책임 위임)
6. `Damage/Status/Spawn` 처리 완료 후 이벤트 발행
7. `CooldownService`가 스킬 상태 갱신

## 5) 설정 파일 예시(의사코드)

```yaml
boss: "boss_iron_hulk"
phases:
  - id: phase_1
    hpRange: [0.67, 1.0]
    skillWeights:
      projectile_combo: 30
      quake_field: 20
      pull_and_strike: 5
  - id: phase_2
    hpRange: [0.33, 0.67]
    skillWeights:
      projectile_combo: 45
      zone_burst: 25
      chain_stun: 10
skills:
  - id: projectile_combo
    effects:
      - type: ProjectileEffect
        count: 5
        spreadDeg: 25
        speed: 9
      - type: DamageEffect
        amount: 18
      - type: StatusEffect
        stun: 0.3
  - id: zone_burst
    effects:
      - type: AreaDamageEffect
        radius: 3.0
        duration: 1.8
      - type: DamageEffect
        amount: 14
```

## 6) 운영/빌드 플로우(권장)
- 밸런스 값 변경은 `BossConfigSO`와 `SkillEffectSO` 수정
- 위험 지표는 `SkillAttempt`, `CastFail`, `TargetMissing`, `HitSuccess`, `PlayerDeathTimeGap` 로그 기준 수치화
- CI 단계에서 SO 스키마 검증(필수 필드, 범위, 중복 id, null 참조) 실행

## 7) 전용 개발 체크리스트
- `[ ]` 스킬 1회 실행 시 데미지/이팩트 누락 없음
- `[ ]` 타깃이 없는 상태에서도 프레임 드랍/예외 없이 실패 처리
- `[ ]` 보스 사망/페이즈 전환 간 스킬 코루틴 안전 종료
- `[ ]` 동일 프레임 다중 타겟 피해 시 중복 판정 기준 일관
- `[ ]` SO 변경이 런타임 참조 깨짐 없이 반영되는지 검증
