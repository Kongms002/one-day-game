# One-Day Game 기획안

## 1. 게임 개요
- 장르: 탑다운 2D 서바이벌 액션 + 빌드/강화형 무기 시스템
- 플랫폼: 모바일 우선(조이스틱 기반), PC 대응
- 목표: **스테이지 100** 클리어 또는 리더보드형 점수 최대화
- 진행: 100 스테이지, 10단위 구간 보스 등장
- 핵심 재미: "근거리/투사체/영역/지속딜 무기 조합 + 속성 대응 + 증강 메커니즘"

---

## 2. 게임 루프
1. 플레이어가 적을 처치해 XP 획득
2. XP 근거리 자석 획득으로 무기 강화 후보 생성
3. 증강 선택(한 번 선택하면 되돌릴 수 없음)
4. 무기 4칸 슬롯 확장, 조합 효율 극대화
5. 10스테이지마다 보스전과 맵/난이도 변화
6. 100스테이지까지 생존 및 점수 갱신 후 종료 화면

- 게임 오버: 체력 0
- 엔딩/재시작: 실패 시에도 **Score, Run Data, 재시작 버튼** 표시

---

## 3. 스테이지/맵 구조
- 스테이지는 경험치 레벨로 간주
- 총 100단계
- 매 10단계마다 보스 라운드 발생
- 스테이지 증가에 따른 난이도 요소
  - 적 체력/속도
  - 적군 구성 비율
  - 특수 적 비율 증가
  - 보스 패턴 강화
- 맵은 구간별로 변경(타일셋/장식/생존 동선/장해물 차등)

---

## 4. 플레이어/전투 시스템
- 이동
  - 기본 이동 + 경계 clamp
  - 이동 속도는 아이템/증강으로 증가 가능
- 근접 접촉 대미지(지속/쿨다운 기반)
- 기본 사망 조건: HP 0
- 궁극기
  - 자원형(상태/쿨다운 연동)
  - 무기별 최종 터치 수치로 강화

---

## 5. 무기 시스템 (핵심)
### 5.1 무기 슬롯 규칙
- 최대 장착 수: **4개**
- 장착 슬롯: `W1~W4`
- 새 무기 획득 시: 빈 슬롯 우선, 꽉 차면 변경 불가(또는 현재 룰 기준 유지)
- 증강은 **한 번 받으면 영구 고정** (제거/회수 불가)
- 무기 레벨: 1~6, 레벨별 배율 적용 (최종 +150% 가이드)

### 5.2 무기 타입
1) 회전 무기
- 근접형 범위 공격(회전 판정)
- 옵션: 회전 속도, 범위, 관통/체력 비례 피해, 느려짐/기절 연계

2) 투사체 무기
- 화살
  - 고정 방향형 기본
  - 업그레이드로 추적형 분기 가능
- 부메랑
  - 발사 후 귀환
  - 다단 명중 가능
- 포탑
  - 소환형/자체 공격 간격 관리
  - 어그로 기반 우선 타격

3) 영역 전개 무기
- 장판
  - 접촉/체류형 지속딜
- 블랙홀
  - 흡입 + 주변 피해
- 독구름
  - DOT(독 데미지) 적용(비치명성: HP 1 보장 예외)

4) 지속딜 무기(펫형)
- 레이저 펫: 가장 가까운 적 우선 정밀 사격
- 광란 펫: 주변 적 유도/상호 공격 유도
- 기절 펫: 가장 가까운 적 기절/제어

### 5.3 발사 방식 분기
- `방향 고정형`
  - 캐릭터 진행/조준 방향으로 발사
  - 예측각/산탄/속도 기반 보정
- `자동 가까운 적 추적형`
  - 최단거리 우선, LOS 및 재탐색 주기 존재
  - 목표 상실 시 대기/회수/재할당 동작 정의

---

## 6. 속성 시스템
- 화염: DOT 지속 피해 + 상태 지속시간 증가
- 독: DOT, 다만 단독 치명 데미지로만은 킬 불가(최저 생명 보장)
- 얼음: 이동 둔화 + 짧은 기절
- 번개: 즉시 기절 + splash 전기 감전
- 시너지 룰: 속성 간 결합 보정 가능(예: 독+블랙홀, 화염+장판)

---

## 7. 증강 시스템
- 증강은 런당 정해진 횟수만큼만 선택
- 증강 카테고리
  - 무기 추가(빈 슬롯/교체 불가 정책 충돌 시 보류)
  - 무기 업그레이드
  - 공격속도 증가
  - 투사체 크기 증가
  - 데미지 증가
  - 이동 속도 증가
- 증강 규칙
  - 즉시 반영, 영구 적용
  - 이미 같은 증강 중복 시 가중 규칙 적용(예: 효율 감소 또는 단계적 강화)
  - 제거/환불 없음

---

## 8. 무기 조합(시너지) 규칙
- 장착 무기 4개 중 **2종 조합(C2)당 6개 시너지 슬롯** 검사
- 시너지는 양쪽 무기가 최대로 강화된 상태에서만 발동
- 시너지 예시
  - 회전 + 투사체: 근접 집결 시 투사체 유효거리 보정
  - 회전 + 블랙홀: 둔화 상태 대상 추가 데미지
  - 투사체 + 포탑: 명중 후 분산/관통 보정
  - 장판 + 독구름: DOT 중첩 효율 상승
  - 펫류 + 기절/독: 제어 지속시간 증가

---

## 9. 적/보스 디자인
### 적 타입
- 자폭형: 접촉/범위 폭발
- 고속형: 추격/회피 패턴
- 증식형: 시간/체력 조건 분열
- 군집형: 한 방향 밀집 돌입
- 원거리형: 제한적 배치, 보스와 비슷한 패턴 변주 대응

### 보스 규칙
- 10/20/30/…/100 보스 구간마다 등장
- 보스는 최소 2개 이상의 무기 축(투사체+영역, 근접+제어 등) 보유 느낌으로 설계
- 3단계 페이즈 전환(HP 임계치 기반)
- 보스별 약점: 특정 무기 타입/속성 상성 적용

## 10. 점수/엔딩
- 점수 반영 요소
  - 스테이지 도달치
  - 처치 수
  - 보스 클리어 횟수
  - 생존 시간
  - 무기 조합 효율/시너지 사용량
- 종료 화면
  - 총점, 스테이지, 리워드 요약, 사용 무기, 사용 증강 목록
  - 버튼: 재시작 / 메인

## 11. 구현 아키텍처 (Unity)
### 현재 구조와 확장 방향
- 현재 단일 무기 정책(`IWeaponPolicy`)에서 다중 무기 시스템으로 분리
- 추천 구조
  - `WeaponTemplateSO`(데이터)
  - `WeaponInstance`(레벨/강화 상태)
  - `WeaponLoadout`(최대 4개)
  - `WeaponBehavior`(실행)
  - `TargetingStrategy`(고정/추적)
  - `UpgradeService`(영구 증강 처리)

### 디렉터리 예시
- `Assets/Scripts/Domain/Weapon`
  - `WeaponTemplate.cs`, `WeaponInstance.cs`, `WeaponSlot.cs`, `WeaponSynergyRule.cs`
- `Assets/Scripts/Application/Weapon`
  - `WeaponLoadoutService.cs`, `WeaponUpgradeService.cs`
- `Assets/Scripts/Presentation/Gameplay/Weapons`
  - `WeaponOrchestrator.cs`, 각 무기 동작 스크립트

### SOLID 적용
- SRP: 무기 판정/이동/표시/레벨 계산 분리
- OCP: 새 무기 타입은 스크립트 추가 + SO 등록만으로 확장
- LSP: 모든 무기 동작은 공통 인터페이스 준수
- ISP: 공격/시각/상태계열 인터페이스 분리
- DIP: 랜덤/입력/시간/풀링 서비스를 인터페이스로 주입

## 12. 단계별 개발 계획
### MVP(최소 실행)
1. 무기 슬롯 4개 기반 도입
2. 기존 1종 무기 -> 4개 슬롯 동작 파이프라인
3. 고정 발사/추적 발사만 분기
4. 증강 영구 선택 기능

### v1.1
1. 투사체 3종(화살/부메랑/포탑)
2. 영역 2종(장판/독구름)
3. 기본 보스 구간 추가

### v1.2
1. 시너지 시스템
2. 보스 패턴 확장
3. 점수/랭킹/재시작 UI 정교화

---

## 13. 밸런스 가이드(초기 권장)
- 성장 곡선은 스테이지 초기 1~40은 완만, 중후반 40~90은 기울기 증가
- 무기당 레벨상한은 6
- 동일 계열 과도한 집중 방지 위해 각 무기 계열별 과금/강화 하한치 존재
- 독은 비치명성 보장 규칙으로 생존 난이도 보정
